# A placeholder post

This file exists at `content/writing/welcome.mdx`. It's here so the writing list has something to render alongside the Medium articles, and so the on-site publishing flow can be verified end-to-end.

## How to publish a new essay

1. Create a new file at `content/writing/your-slug.mdx`.
2. Add an `export const metadata = { ... }` block at the top with three fields: `title`, `date` (YYYY-MM-DD), `description`.
3. Add the slug, title, date, and description to the `internalPosts` array in `src/lib/writing.ts`.
4. Write the body in Markdown.
5. Save. The dev server picks it up automatically — refresh the page.

## What works in posts

Markdown basics — **bold**, *italic*, [links](https://example.com), inline `code`, and lists:

- Bullet points work.
- So does nesting and ordering.
- `code spans` render in a monospace font.

Block quotes carry the accent color:

> The mind that thinks in spirals is the mind that finds the seam between what was and what could be.

Horizontal rules render as hairline dividers.

---

Code blocks render as boxed monospace text:

```ts
function helloWorld() {
  return "hello, world";
}
```

That's the whole feature surface. Delete this file when you publish your first real essay.
